WHAT EACH TOOL DOES
================================================================================

CONNECTIVITY_DIAGNOSTICS.PY
Purpose: Test network and WebSocket connectivity
Usage: python tools/connectivity_diagnostics.py [--host HOST] [--port PORT] [--json FILE] [--html FILE]
Time: ~45-60 seconds (includes 30s flakiness test)
Output: Console summary, optional JSON/HTML exports
When to use: Troubleshoot connection issues, verify network setup

SERVER_HEALTH_CHECK.PY
Purpose: Validate render server configuration
Usage: python tools/server_health_check.py [--blender PATH] [--json FILE]
Time: ~30-60 seconds (includes GPU benchmark)
Output: Console summary, readiness score, optional JSON export
When to use: Ensure server is ready to render, diagnose render failures

test_addon_connection.py
Purpose: Automated testing of addon components
Usage: pytest tests/test_addon_connection.py -v
Time: ~5-10 seconds
Output: Test results, pass/fail summary
When to use: Verify addon code quality, test before commits

logging_system.py
Purpose: Structured logging for addon events
Usage: from addon.logging_system import get_logger
Time: Minimal (background operation)
Output: JSON logs to ~/.blender/remote-gpu/logs/
When to use: Debug issues, track events, generate bug reports

tests.yml (GitHub Actions)
Purpose: Automated CI/CD testing
Usage: Automatic (runs on every commit)
Time: ~5 minutes per build
Output: Pass/fail status in GitHub Actions tab
When to use: Ensure code quality on every commit

INTEGRATION WITH BUG REPORTING
================================================================================

When user reports a bug, they can:

1. Run connectivity test
   $ python tools/connectivity_diagnostics.py --json conn.json --html conn.html

2. Run server health check
   $ python tools/server_health_check.py --json health.json

3. Export addon logs
   From Blender console:
   from addon.logging_system import get_logger
   logger = get_logger()
   logger.export_session_log()

4. Attach files to GitHub issue:
   - conn.json (machine-readable diagnostics)
   - conn.html (formatted for human review)
   - health.json (server validation)
   - session_export_*.json (addon event logs)
   - Steps to reproduce
   - Expected vs actual behavior

Developer receives complete diagnostic data for immediate troubleshooting.
