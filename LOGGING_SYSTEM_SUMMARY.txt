ERROR CODES (40+ codes)
================================================================================

CATEGORIES:
  CONN_*  (6 codes)   Connection errors (timeout, refused, lost, upgrade failed)
  SYNC_*  (6 codes)   Scene sync errors (corrupt, missing textures, oversized)
  RENDER_*(7 codes)   Render errors (load failed, GPU unavailable, timeout)
  SCENE_* (4 codes)   Scene validation (invalid geometry, missing files, no camera)
  GPU_*   (5 codes)   GPU hardware (out of VRAM, driver crash, thermal throttle)
  ENCODE_*(4 codes)   Encoding errors (JPEG/EXR decode failed)
  PROTO_* (4 codes)   Protocol errors (version mismatch, invalid format)
  SYS_*   (5 codes)   System errors (missing deps, Blender not found, disk full)
  PERF_*  (4 codes)   Performance (slow network, high latency, low fps)
  AUTH_*  (3 codes)   Auth/permissions (permission denied, addon disabled)
  INFO_*  (3 codes)   Success messages (connected, synced, render complete)

EACH ERROR INCLUDES:
  - Code ID (e.g., CONN_001)
  - Technical description
  - User-friendly message
  - Recovery suggestions (list of steps to try)
  - Severity level (CRITICAL/ERROR/WARNING/INFO)

================================================================================
FEATURES
================================================================================

1. STRUCTURED JSON LOGGING
   - Optional JSON output for automated parsing
   - Includes operation ID, session ID, duration, metrics
   - Log rotation by file size (default 10MB)
   - Configurable output: console, file, or both

2. OPERATION TRACKING
   - Every render operation gets unique ID
   - Every client session gets unique ID
   - Trace entire operation from start to finish
   - Performance metrics logged automatically

3. USER-FRIENDLY ERRORS
   - Technical errors mapped to clear messages
   - Recovery suggestions shown in Blender UI
   - Full stack trace logged internally, hidden from user
   - Error tracking per session

4. PERFORMANCE MONITORING
   - Automatic timing for all major operations
   - Network metrics (upload speed, latency)
   - GPU metrics (VRAM usage, temperature)
   - Frame metrics (encoding time, decode time)

5. FULL TRACEABILITY
   - Every log entry has: timestamp, level, module, function, line number
   - Can trace a render from Blender → network → server → GPU → back to Blender
   - Session ID links all operations from one client
   - Operation ID links all steps within one operation

SERVER STARTUP
================================================================================

BASIC:
  python server.py --port 9876

WITH LOGGING:
  python server.py --port 9876 --log-file server.log

DEBUG MODE (JSON format):
  python server.py --log-level DEBUG --json-logs --log-file server.json.log

CUSTOM BLENDER PATH:
  python server.py --blender /path/to/blender --log-file server.log

================================================================================
VIEWING LOGS
================================================================================

CONSOLE (real-time):
  tail -f server.log
  tail -f server.log | grep ERROR

ANALYSIS:
  grep "CONN_001" server.log              (find specific error)
  grep "sess_abc1" server.log             (trace session)
  grep "op=render_op1" server.log         (trace operation)
  
JSON ANALYSIS:
  jq 'select(.level == "ERROR")' server.json.log
  jq 'select(.error_code == "RENDER_001")' server.json.log
  jq '.metrics' server.json.log
