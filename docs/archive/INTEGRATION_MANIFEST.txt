  0x21 VIEWPORT_STOP           Stop viewport render

  0x23 TIMELINE_UPDATE         Frame/play state (Agent 3)

  0x30 BAKE_START              Start bake job
  0x40 SIM_START               Start simulation
  0xF0 PING                    Heartbeat

  0x81 FRAME_FINAL             EXR final render (with frame metadata, Agent 3)

  0x90 STATUS                  Server status (with GPU info, Agent 4)
  0x91 SCENE_ACK               Scene received
  0x92 DELTA_ACK               Delta applied
  0xF1 PONG                    Heartbeat response

================================================================================
USAGE EXAMPLES
================================================================================

AGENT 1 - DENOISER SELECTION:
  User opens Blender Render Properties → Remote GPU
  Sets: Denoiser = "OptiX", Intensity = 0.8
  Clicks Render → addon sends RENDER_SETTINGS to server
  Server generates Blender script with AOV creation + OptiX denoising
  Result: High-quality render with AI noise reduction

AGENT 2 - FRAME BUFFERING:
  During 30fps viewport render, server generates 32 frames/sec
  Client RingBuffer holds last 50 frames (prevents memory leak)
  Old frames auto-dropped when buffer full
  Latency tracked: if frame >500ms old, client skips to latest
  Result: Smooth 30fps playback even with network variation

AGENT 3 - ANIMATION PLAYBACK:
  User plays timeline in Blender (25 frames/sec animation)
  Addon detects frame changes, sends TIMELINE_UPDATE for each frame
  Server stores current_frame in _timeline_state
  Renderer uses frame number for correct animation keyframes
  Result: Frame-accurate multi-frame animation rendering

AGENT 4 - GPU BACKEND SWITCHING:
  Server detects available backends: OptiX, CUDA
  Sends STATUS with available_backends = ["OPTIX", "CUDA"]
  User clicks "Switch to CUDA" in Blender
  Client sends RENDER_DEVICE_SELECT {"backend": "CUDA"}
  Server validates, updates manager, next render uses CUDA
  Result: Runtime GPU selection without restart
